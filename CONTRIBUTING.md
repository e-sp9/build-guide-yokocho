# Contributing — 自作キーボード ビルドガイド横丁

ビルドガイド横丁は **PRドリブン** で運営されています。
あなたのキーボードを掲載したい、誤りを直したい、サイト本体を改善したい — どれも歓迎します。

## TL;DR — 自分のキーボードを掲載したい

1. このリポジトリを fork する
2. `src/content/guides/<your-keyboard-id>.json` を新規作成する
3. PRを送る (PRテンプレートに従って情報を埋める)

それだけです。

---

## ガイドJSONのスキーマ

`src/content/guides/<id>.json` は次のフィールドを持つJSONファイルです。
ファイル名 (`<id>.json`) はそのままURL等で使われる識別子になります。半角英数字とハイフンのみ、小文字推奨。

```json
{
  "name": "Corne",
  "designer": "foostan",
  "description": "3x6カラムスタッガード+3キーの分割キーボード。CRKBDの愛称で親しまれ、自作キーボード入門の代表格として国内外に愛用者が多い。",
  "url": "https://github.com/foostan/crkbd",
  "tags": ["分割", "40%", "カラムスタッガード", "MX互換", "Choc互換"],
  "shopLinks": [
    { "name": "遊舎工房", "url": "https://shop.yushakobo.jp/products/corne-cherry" }
  ],
  "addedAt": "2026-04-12"
}
```

| フィールド    | 必須 | 内容                                                                  |
| ------------- | ---- | --------------------------------------------------------------------- |
| `name`        | ✓    | キーボードの名称                                                      |
| `designer`    | ✓    | 設計者名 (GitHubハンドル / Twitterハンドルなど)                       |
| `description` | ✓    | 280文字以内のキャッチーな説明文                                       |
| `url`         | ✓    | ビルドガイドのURL (作者サイト・GitHub READMEなど)                     |
| `tags`        | ✓    | タグの配列。既存タグを優先して使う                                    |
| `shopLinks`   | -    | `{ name, url }` の配列。複数のショップリンクを登録可能                |
| `addedAt`     | ✓    | このエントリを追加した日付 (ISO 8601 / `YYYY-MM-DD`)。並び順に使われる |

スキーマは [`src/content.config.ts`](./src/content.config.ts) で Zod により厳密に検証されます。
不正なJSONは `pnpm build` で即座にエラーになります。

## タグの方針

- まずは [`src/content/guides/`](./src/content/guides/) 内の既存JSONを眺めて、流用できるタグを探してください
- 新しいタグを追加する場合は、抽象度を既存タグと揃えてください
  - 良い例: `分割`, `40%`, `カラムスタッガード`, `Choc互換`, `OLED`, `トラックボール`
  - 微妙な例: `自作キーボード` (全部該当する), `かっこいい` (主観)

## 掲載許可について

ビルドガイド横丁はリンク集の性格を持つため、URLの紹介自体は問題ないと考えています。
ただし、設計者への敬意を最優先するため、以下のいずれかを満たしてください。

- あなたが設計者本人である
- 設計者から掲載許可を得ている (PR本文に連絡日などを記載)
- 設計者と連絡がつかない場合、公開情報のみに基づくリンク紹介として掲載

掲載後に設計者から取り下げの希望があれば、速やかに対応します。

## ローカルでの動作確認

```bash
pnpm install
pnpm dev      # http://localhost:4321/ で確認
pnpm build    # スキーマバリデーション + 静的ビルド
```

`pnpm build` が通れば、CIも通ります。

## サイト本体の改修

UI、フィルタ、検索、デザイン、SEOなど、サイト本体への改善PRも歓迎します。
大きめの変更は事前にIssueで相談してもらえると進めやすいです。
